# 模块注册中心 (Module Registry)

## 概述

模块注册中心是一个用于统一管理 `src/modules` 下所有子模块资源的工具。它提供了一个集中式的注册机制，让各个子模块可以将 API、组件、工具函数、静态资源等注册为全局可访问的资源。

## 特性

- 🎯 **统一管理**: 集中管理所有模块的资源
- 🔧 **类型安全**: 完整的 TypeScript 类型支持
- 🚀 **灵活注册**: 支持单个资源或批量资源注册
- 🔍 **便捷查询**: 提供多种资源查询方式
- 🎨 **链式调用**: 支持链式调用的构建器模式
- 📦 **自动发现**: 支持根据目录结构自动注册资源

## 基础用法

### 1. 注册模块资源

#### 方式一：直接注册完整模块

```typescript
import { moduleRegistry } from '@/utils/module-registry';

// 注册设备管理模块
moduleRegistry.register('device-manager', {
  moduleId: 'device-manager',
  apis: {
    getDeviceList: async (params: any) => {
      // API 实现
      return { data: [] };
    },
    createDevice: async (device: any) => {
      return { success: true };
    }
  },
  components: {
    DeviceTable: defineAsyncComponent(() => import('./components/DeviceTable.vue')),
    DeviceForm: defineAsyncComponent(() => import('./components/DeviceForm.vue'))
  },
  utils: {
    formatDeviceStatus: (status: string) => {
      const statusMap: Record<string, string> = {
        'online': '在线',
        'offline': '离线'
      };
      return statusMap[status] || '未知';
    }
  }
});
```

#### 方式二：分别注册不同类型的资源

```typescript
// 注册 API
moduleRegistry.registerResource('device-manager', 'apis', {
  getDeviceList: async (params: any) => ({ data: [] }),
  createDevice: async (device: any) => ({ success: true })
});

// 注册组件
moduleRegistry.registerResource('device-manager', 'components', {
  DeviceTable: defineAsyncComponent(() => import('./components/DeviceTable.vue'))
});

// 注册工具函数
moduleRegistry.registerResource('device-manager', 'utils', {
  formatDeviceStatus: (status: string) => status
});
```

#### 方式三：使用构建器模式

```typescript
import { createModuleBuilder } from '@/utils/module-register-helper';

createModuleBuilder('device-manager')
  .apis({
    getDeviceList: async (params: any) => ({ data: [] })
  })
  .components({
    DeviceTable: defineAsyncComponent(() => import('./components/DeviceTable.vue'))
  })
  .utils({
    formatDeviceStatus: (status: string) => status
  })
  .override() // 允许覆盖已存在的资源
  .register(); // 执行注册
```

#### 方式四：快速注册

```typescript
import { quickRegister } from '@/utils/module-register-helper';

quickRegister('device-manager', {
  apis: {
    getDeviceList: async (params: any) => ({ data: [] })
  },
  components: {
    DeviceTable: defineAsyncComponent(() => import('./components/DeviceTable.vue'))
  },
  utils: {
    formatDeviceStatus: (status: string) => status
  }
});
```

### 2. 获取已注册的资源

#### 获取完整模块

```typescript
const deviceModule = moduleRegistry.getModule('device-manager');
console.log(deviceModule); // 完整的模块资源对象
```

#### 获取特定类型的资源

```typescript
// 获取所有 API
const apis = moduleRegistry.getResource('device-manager', 'apis');

// 获取所有组件
const components = moduleRegistry.getResource('device-manager', 'components');

// 获取所有工具函数
const utils = moduleRegistry.getResource('device-manager', 'utils');
```

#### 获取具体的资源项

```typescript
// 获取特定的 API
const getDeviceList = moduleRegistry.getResourceItem('device-manager', 'apis', 'getDeviceList');
if (getDeviceList) {
  const result = await getDeviceList({ page: 1, size: 10 });
}

// 获取特定的工具函数
const formatStatus = moduleRegistry.getResourceItem('device-manager', 'utils', 'formatDeviceStatus');
if (formatStatus) {
  console.log(formatStatus('online')); // 在线
}

// 获取特定的组件
const DeviceTable = moduleRegistry.getResourceItem('device-manager', 'components', 'DeviceTable');
```

### 3. 检查资源是否存在

```typescript
// 检查模块是否存在
if (moduleRegistry.hasModule('device-manager')) {
  console.log('设备管理模块已注册');
}

// 检查特定资源类型是否存在
if (moduleRegistry.hasResource('device-manager', 'apis')) {
  console.log('设备管理模块有API资源');
}

// 检查具体资源项是否存在
if (moduleRegistry.hasResourceItem('device-manager', 'apis', 'getDeviceList')) {
  console.log('getDeviceList API 已注册');
}
```

## 在 Vue 组件中使用

### 1. 基础用法

```vue
<template>
  <div>
    <device-table :data="devices" />
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { moduleRegistry } from '@/utils/module-registry';

const devices = ref([]);

onMounted(async () => {
  // 获取设备列表 API
  const getDeviceList = moduleRegistry.getResourceItem('device-manager', 'apis', 'getDeviceList');
  
  if (getDeviceList) {
    const result = await getDeviceList({ page: 1, size: 10 });
    devices.value = result.data;
  }
});

// 动态加载组件
const DeviceTable = moduleRegistry.getResourceItem('device-manager', 'components', 'DeviceTable');
</script>
```

### 2. 创建 Composable

```typescript
// composables/useDeviceModule.ts
import { moduleRegistry } from '@/utils/module-registry';

export function useDeviceModule() {
  // 获取设备模块的所有资源
  const deviceModule = moduleRegistry.getModule('device-manager');
  
  // 创建API调用函数
  const getDeviceList = async (params: any) => {
    const api = moduleRegistry.getResourceItem('device-manager', 'apis', 'getDeviceList');
    if (!api) throw new Error('getDeviceList API 未找到');
    return await api(params);
  };

  const createDevice = async (device: any) => {
    const api = moduleRegistry.getResourceItem('device-manager', 'apis', 'createDevice');
    if (!api) throw new Error('createDevice API 未找到');
    return await api(device);
  };

  // 获取工具函数
  const formatStatus = moduleRegistry.getResourceItem('device-manager', 'utils', 'formatDeviceStatus');

  return {
    deviceModule,
    getDeviceList,
    createDevice,
    formatStatus
  };
}
```

在组件中使用：

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { useDeviceModule } from '@/composables/useDeviceModule';

const { getDeviceList, formatStatus } = useDeviceModule();
const devices = ref([]);

onMounted(async () => {
  const result = await getDeviceList({ page: 1, size: 10 });
  devices.value = result.data.map((device: any) => ({
    ...device,
    statusText: formatStatus?.(device.status) || device.status
  }));
});
</script>
```

## 高级功能

### 1. 批量注册

```typescript
const modules = [
  {
    moduleId: 'authentication',
    resource: {
      apis: {
        login: async (credentials: any) => ({ token: 'mock-token' }),
        logout: async () => ({ success: true })
      }
    }
  },
  {
    moduleId: 'notification',
    resource: {
      components: {
        NotificationCenter: defineAsyncComponent(() => import('./components/NotificationCenter.vue'))
      }
    }
  }
];

moduleRegistry.batchRegister(modules, {
  allowPartialFailure: true // 允许部分注册失败
});
```

### 2. 搜索功能

```typescript
// 搜索所有包含 APIs 的模块
const modulesWithApis = moduleRegistry.searchModules('apis');
console.log('包含API的模块:', modulesWithApis);

// 搜索包含特定API的模块
const modulesWithLogin = moduleRegistry.searchModules('apis', 'login');
console.log('包含login API的模块:', modulesWithLogin);

// 获取所有已注册的模块ID
const allModules = moduleRegistry.getAllModuleIds();
console.log('所有已注册模块:', allModules);
```

### 3. 自动注册

```typescript
import { autoRegisterModule } from '@/utils/module-register-helper';

// 根据目录结构自动注册模块
await autoRegisterModule('device-manager', '/src/modules/device-manager-ui', {
  override: true,
  allowPartialFailure: true
});
```

### 4. 资源管理

```typescript
// 移除整个模块
moduleRegistry.unregister('device-manager');

// 移除模块的特定资源类型
moduleRegistry.unregisterResource('device-manager', 'apis');

// 清空所有模块
moduleRegistry.clear();
```

## 最佳实践

### 1. 命名规范

- **模块ID**: 使用 kebab-case，如 `device-manager`、`data-collector`
- **资源名称**: 使用 camelCase，如 `getDeviceList`、`DeviceTable`
- **保持一致性**: 在整个项目中保持命名风格的一致性

### 2. 类型安全

```typescript
// 定义强类型的API接口
interface DeviceAPI {
  getDeviceList(params: { page: number; size: number }): Promise<{ data: Device[] }>;
  createDevice(device: CreateDeviceRequest): Promise<{ success: boolean }>;
}

// 注册时使用类型断言
moduleRegistry.registerResource('device-manager', 'apis', apis as DeviceAPI);

// 使用时进行类型检查
const api = moduleRegistry.getResourceItem<DeviceAPI['getDeviceList']>('device-manager', 'apis', 'getDeviceList');
```

### 3. 错误处理

```typescript
// 安全的资源获取
function safeGetResource<T>(moduleId: string, resourceType: keyof ModuleResource, resourceName: string): T | null {
  try {
    const resource = moduleRegistry.getResourceItem<T>(moduleId, resourceType, resourceName);
    return resource || null;
  } catch (error) {
    console.error(`获取资源失败: ${moduleId}.${String(resourceType)}.${resourceName}`, error);
    return null;
  }
}

// 使用
const api = safeGetResource<Function>('device-manager', 'apis', 'getDeviceList');
if (api) {
  const result = await api(params);
}
```

### 4. 开发调试

```typescript
// 开发模式下启用详细日志
if (import.meta.env.DEV) {
```

## API 参考

### ModuleRegistry 类

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `register` | 注册模块资源 | `(moduleId, resource, options?)` | `void` |
| `registerResource` | 注册特定类型资源 | `(moduleId, resourceType, resources, options?)` | `void` |
| `getModule` | 获取完整模块 | `(moduleId)` | `ModuleResource \| undefined` |
| `getResource` | 获取特定类型资源 | `(moduleId, resourceType)` | `Record<string, T> \| undefined` |
| `getResourceItem` | 获取具体资源项 | `(moduleId, resourceType, resourceName)` | `T \| undefined` |
| `hasModule` | 检查模块是否存在 | `(moduleId)` | `boolean` |
| `hasResource` | 检查资源类型是否存在 | `(moduleId, resourceType)` | `boolean` |
| `hasResourceItem` | 检查具体资源项是否存在 | `(moduleId, resourceType, resourceName)` | `boolean` |
| `unregister` | 移除模块 | `(moduleId)` | `boolean` |
| `unregisterResource` | 移除特定类型资源 | `(moduleId, resourceType)` | `boolean` |
| `getAllModuleIds` | 获取所有模块ID | `()` | `string[]` |
| `getAllModules` | 获取所有模块 | `()` | `Map<string, ModuleResource>` |
| `clear` | 清空所有模块 | `()` | `void` |
| `batchRegister` | 批量注册模块 | `(modules, options?)` | `void` |
| `searchModules` | 搜索模块 | `(resourceType, resourceName?)` | `string[]` |

### ModuleRegistryBuilder 类

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `apis` | 添加API资源 | `(apis)` | `this` |
| `components` | 添加组件资源 | `(components)` | `this` |
| `utils` | 添加工具函数资源 | `(utils)` | `this` |
| `assets` | 添加静态资源 | `(assets)` | `this` |
| `hooks` | 添加钩子函数资源 | `(hooks)` | `this` |
| `routes` | 添加路由资源 | `(routes)` | `this` |
| `locales` | 添加国际化资源 | `(locales)` | `this` |
| `stores` | 添加状态管理资源 | `(stores)` | `this` |
| `custom` | 添加自定义资源 | `(key, value)` | `this` |
| `withOptions` | 设置注册选项 | `(options)` | `this` |

| `register` | 执行注册 | `()` | `void` |
| `build` | 构建资源对象 | `()` | `Partial<ModuleResource>` |

## 故障排除

### 常见问题

1. **模块注册失败**
   - 检查模块ID是否为空或重复
   - 确认是否设置了 `override: true` 来覆盖已存在的模块

2. **获取资源失败**
   - 确认模块和资源是否已正确注册
   - 检查模块ID和资源名称拼写是否正确

3. **类型错误**
   - 确保导入了正确的类型定义
   - 检查 TypeScript 配置是否正确

4. **动态导入失败**
   - 确认文件路径是否正确
   - 检查 Vite/Webpack 配置是否支持动态导入
